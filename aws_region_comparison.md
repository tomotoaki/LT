# AWS 東京リージョン vs 大阪リージョン 比較

AWSの東京リージョン（ap-northeast-1）と大阪リージョン（ap-northeast-3）の主な違いをまとめました。

## 概要

| 項目                            | 東京リージョン (ap-northeast-1) | 大阪リージョン (ap-northeast-3)                          |
| :------------------------------ | :------------------------------ | :------------------------------------------------------- |
| **開設年**                      | 2011年                          | 2018年 (ローカルリージョン)<br>2021年 (フルリージョン化) |
| **アベイラビリティゾーン (AZ)** | **4**                           | **3**                                                    |
| **サービス数 (概算)**           | 約 333                          | 約 202                                                   |
| **主な用途**                    | メインシステム、最新機能の利用  | 災害対策 (DR)、西日本エリアへの低遅延配信                |

## 主な違い

### 1. サービスの網羅性
東京リージョンは、AWSのほぼ全てのサービスが利用可能です。新しいサービスや機能は、通常東京リージョンで先行してリリースされます。
大阪リージョンは主要なサービス（EC2, S3, RDS, Lambda, EKSなど）は網羅していますが、一部の専門的なサービスや最新のサービスは未提供の場合があります。

### 2. サービス別対応状況 (主な差異)

| サービスカテゴリ     | サービス名                  | 東京 (ap-northeast-1) | 大阪 (ap-northeast-3) | 備考                                      |
| :------------------- | :-------------------------- | :-------------------: | :-------------------: | :---------------------------------------- |
| **基本サービス**     | EC2, S3, RDS, Lambda, EKS等 |           ○           |           ○           | 主要サービスは両リージョンで利用可能      |
| **生成AI**           | Amazon Bedrock              |           ○           |           ○           | 大阪は2025年2月より対応                   |
| **データベース**     | Amazon Neptune              |           ○           |           ○           | グラフデータベース                        |
| **データベース**     | Amazon Timestream           |           ○           |           ×           | 時系列データベース                        |
| **検索**             | Amazon Kendra               |           ○           |           ×           | インテリジェント検索                      |
| **ブロックチェーン** | Amazon Managed Blockchain   |           ○           |           ×           |                                           |
| **衛星**             | AWS Ground Station          |           ×           |           ×           | 日本国内リージョンでは未提供              |
| **分析**             | Amazon Textract             |           ×           |           ×           | 日本国内リージョンでは未提供 (2025年時点) |

### 3. 大阪リージョンで利用できない、または制限がある主なサービス (詳細)
表に記載した以外にも、以下のサービスなどが大阪リージョンでは未提供、または制限がある場合があります。

*   **Amazon Braket**: 量子コンピューティング
*   **AWS Wavelength**: 5Gエッジコンピューティング (国内未展開)
*   **Amazon HealthLake**: 医療データ分析
*   **AWS IoT RoboRunner**: ロボットフリート管理
*   **Amazon Monitron**: 産業用機器の予知保全
*   **Amazon Location Service**: 一部機能に制限がある場合あり
*   **AWS Outposts**: 一部構成のみ対応

### 4. アベイラビリティゾーン (AZ) の数
*   **東京**: 4 AZ
*   **大阪**: 3 AZ
東京の方がAZが多いため、より高い冗長性を確保したい場合や、特定のAZ障害時の退避先確保において選択肢が多いです。ただし、大阪の3 AZも標準的な高可用性構成には十分です。

## 推奨される使い分け

*   **東京リージョン**:
    *   最新のAWSサービスや機能を利用したい場合
    *   最も幅広いサービスラインナップが必要な場合
    *   4つのAZを活用した極めて高い可用性が求められる場合

*   **大阪リージョン**:
    *   東京リージョンの災害対策（DR）サイトとして
    *   西日本のユーザーに対してより低いレイテンシーを提供したい場合
    *   データの国内保持要件を満たしつつ、地理的冗長性を持たせたい場合
